# Roadmap: Version 0.2.5 - Architectural Refinement & Real-time Sync

## Vision

The goal of Version 0.2.5 is to move away from polling for live chat, improve frontend modularity, and finalize the cleanup items identified during the project audit. This version solidifies the foundation for future scaling and real-time interactive features.

## Strategic Pillars

### 1. Real-time Engine Upgrade

Transitioning from polling to **Server-Sent Events (SSE)** will reduce server load and provide a more responsive experience.

- Backend: Implementation of an SSE generator in the chat router.
- Frontend: Migration of the chat polling logic to an EventSource listener.

### 2. State & Persistence

Persistent live sessions across navigation will be handled via a dedicated `ChatContext`.

- Encapsulate session state, message history, and PTT (Push-to-Talk) status.

### 3. Backend Hardening

Optimizing the event loop by moving background tasks (like visitor tracking) to a more robust in-memory queue.

- Sanitize CORS for security.

### 4. Codebase Hygiene

Resolving all remaining audit points to ensure high maintainability.

## Timeline

- **Phase 1: Hygiene & Audit Resolution** (Current)
- **Phase 2: Frontend Refactoring & Hook Encapsulation**
- **Phase 3: SSE Migration & ChatContext implementation**
- **Phase 4: Backend Performance Optimization**
