# Technical Contract: V0.2.5 SSE Migration & ChatContext

- **Requester:** Orchestrator
- **Status:** Completed (2026-05-19)
- **Version:** 0.2.5

## Scope

### 1. Backend SSE Implementation
- **Target**: `backend/routers/chat.py`
- **New Endpoint**: `GET /api/v1/chat/stream/{session_id}`
- **Action**: 
    - Implement a Server-Sent Events (SSE) generator.
    - The generator should yield new messages for the given `session_id`.
    - It should also signal changes in `is_active` status.
    - Use `asyncio.sleep` (e.g., 1s or 2s) to check for new messages in the DB if no real-time trigger (like Pub/Sub) is implemented, but providing a much more "push" feel than polling.
    - **Optimization**: Use a last-seen message ID or timestamp to only stream new data.

### 2. Frontend ChatContext & Provider
- **Target**: `frontend/lib/context/ChatContext.tsx`
- **Action**: 
    - Create a `ChatProvider` to wrap the entire application.
    - Encapsulate `messages`, `isLoading`, `isLive`, and `sessionId`.
    - Move `useLiveChat` logic into the context or have `useLiveChat` consume the context.
    - **Persistence**: Ensure `localStorage` sync is handled globally.

### 3. Frontend SSE Migration
- **Target**: `frontend/lib/hooks/useLiveChat.ts`
- **Action**: 
    - Replace the `setInterval` polling with an `EventSource` listener connected to the new SSE endpoint.
    - Update state based on events received from the stream.
    - Handle reconnection logic and stream cleanup on unmount.

### 4. Cleanup
- **Action**: Deprecate or remove the `/api/v1/chat/sync/{session_id}` polling endpoint once SSE is verified.

## Acceptance Criteria
- [ ] Real-time updates: Assistant messages appear instantly without waiting for a poll interval.
- [ ] Session persistence: Navigating between pages (Home -> About -> Home) preserves the chat state and active live session without re-fetching.
- [ ] SSE stream handles connection drops and reconnects gracefully.
- [ ] `useLiveChat` is simplified by delegating core state to `ChatContext`.
- [ ] Memory/Performance: No memory leaks from unclosed `EventSource` connections.
- [ ] No regression in Telegram relay or fallback AI persona logic.
