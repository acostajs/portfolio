# Technical Contract: Version 0.1.7 - Performance & Security Hardening

## Overview

This contract outlines the remaining tasks for Version 0.1.7, focusing on optimizing frontend performance (reducing bundle size from 1.9MB) and hardening backend security/clarity.

## Objectives

1.  **Performance Optimization**: Reduce the initial bundle size to < 500kB (gzipped) by implementing code splitting and lazy loading.
2.  **Security Hardening**: Clarify admin authentication parameters and implement accessibility-driven motion reduction.
3.  **DX Improvements**: Ensure environment variables and parameters are consistently named.

## Scope of Work

### 1. Frontend: Performance Optimization

- **Code Splitting**:
  - Implement `React.lazy()` for heavy components:
    - `BlogManager.tsx` (uses `@codesandbox/sandpack-react`)
    - `BotMessage.tsx` (uses `react-markdown` and `react-syntax-highlighter`)
  - Use `Suspense` with skeleton loaders for these components.
- **Dependency Audit**:
  - Evaluate if `react-syntax-highlighter` can be replaced with a lighter alternative or if its styles can be lazy-loaded.
  - Analyze the 1.9MB bundle using `rollup-plugin-visualizer`.
- **Resource Priority**:
  - Add `<link rel="preload">` for critical fonts and the `hero.png` asset.
- **Asset Compression**:
  - Optimize `hero.png` and PDF resumes.

### 2. Backend: Security & Clarity

- **Parameter Renaming**:
  - Rename `password` parameter to `admin_token` in `backend/main.py` and `backend/auth.py` for clarity.
  - Update `backend/admin.py` to reflect these changes if applicable.
- **Environment Hardening**:
  - Ensure all sensitive keys are strictly pulled from `.env` (already partially done, verify coverage).

### 3. UX: Accessibility

- **Reduced Motion**:
  - Update `MeshBackground.tsx` to detect `prefers-reduced-motion` and disable animations accordingly.

## Acceptance Criteria

- [ ] Initial bundle size (gzipped) is significantly reduced.
- [ ] No regressions in Admin section or Chatbot functionality.
- [ ] Backend API documentation (Swagger) reflects the renamed `admin_token` parameter.
- [ ] Lighthouse Performance score remains 95+ after optimizations.

## Delegation

- **Frontend Specialist**: Responsible for code splitting, lazy loading, and mesh background updates.
- **Backend Specialist**: Responsible for parameter renaming and security verification.
