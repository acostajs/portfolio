# Technical Contract: v0.1.8 - Cloud Infrastructure & Storage

**Date:** 2026-05-11
**Status:** Approved
**Version:** 0.1.8
**Owner:** Orchestrator

## 1. Objective

Transition image storage from the local filesystem (incompatible with Vercel's ephemeral/read-only filesystem) to **Google Cloud Storage (GCS)**. Enhance environment security by hardening credential management.

## 2. Architectural Changes

### 2.1 Backend (FastAPI)

- **New Dependency:** `google-cloud-storage`.
- **Storage Abstraction:** Implement `backend/storage.py` to handle:
  - `upload_file(file: UploadFile, folder: str) -> str`: Uploads to GCS and returns the public URL.
  - `delete_file(file_url: str)`: Removes the object from GCS.
- **Config Update:** Add GCS-related environment variables to `backend/config.py`:
  - `GOOGLE_SERVICE_ACCOUNT_EMAIL` (Existing)
  - `GOOGLE_PRIVATE_KEY` (Existing)
  - `GCS_BUCKET_NAME` (New)

### 2.2 Database (SQLModel)

- Image paths in `Project` and `Experience` models will now store absolute GCS URLs instead of relative local paths.
- **Migration Strategy:** A script will be needed to upload existing local images to GCS and update DB records.

### 2.3 Frontend (React)

- Components will use absolute URLs provided by the API.
- Ensure `admin/ProjectsManager.tsx` and `admin/ExperienceManager.tsx` handle absolute URLs for image previews.

## 3. API Updates

### 3.1 Admin Endpoints

- `POST /api/v1/admin/projects/upload`: Will now use `storage.py` to pipe data directly to GCS.
- `DELETE /api/v1/admin/projects/{id}/image`: Will call `storage.py.delete_file`.

## 4. Security & Hardening

- **Credentials:** Use existing `GOOGLE_SERVICE_ACCOUNT_EMAIL` and `GOOGLE_PRIVATE_KEY`.
- **Signed URLs vs Public Access:**
  - Portfolio images will be stored in a **publicly readable bucket**.
  - Admin credentials are managed via Vercel's encrypted environment variables.

## 5. Implementation Plan

1. **Phase 1: Backend Utility**
   - Implement `backend/storage.py` using `service_account.Credentials.from_service_account_info`.
   - Update `backend/config.py`.
2. **Phase 2: Integration**
   - Refactor `backend/admin.py` to use GCS for project and experience images.
3. **Phase 3: Migration & Validation**
   - Script to move current `/public/uploads` (if any) to GCS and update DB.
   - Verify image rendering in Frontend.

## 6. Success Criteria

- [ ] Images are successfully uploaded to GCS via the Admin panel.
- [ ] Images are rendered correctly on the Projects/Experience pages using GCS URLs.
- [ ] No local filesystem writes occur during the upload process.
- [ ] GCS credentials are not hardcoded or exposed in logs.
