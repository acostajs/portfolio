# Technical Contract: v0.1.9 Architecture Consolidation

**Status:** Draft
**Owner:** Orchestrator
**Specialist:** frontend-agent

## 1. Context & Vision

The project has grown rapidly, leading to some duplication in data fetching logic and markdown rendering. To ensure scalability and ease of maintenance, we need to unify these core patterns.

## 2. Technical Specifications

### 2.1 Unified Data Fetching

- **Requirement:** Eliminate direct `fetch()` calls in page components for internal API endpoints.
- **Action:**
  - Migrate `About.tsx`, `Experience.tsx`, `Projects.tsx`, and `Blog.tsx` to use `fetchPublic` from `lib/api.ts`.
  - Refactor `Home.tsx` to use standardized API utilities for chat and feedback.
  - Ensure all `fetchPublic` calls are type-safe using the generics provided.

### 2.2 Consolidated Markdown Rendering

- **Requirement:** Use a single, optimized component for all Markdown rendering.
- **Action:**
  - Move `frontend/src/components/chat/MarkdownRenderer.tsx` to `frontend/src/components/ui/SharedMarkdown.tsx`.
  - Update the component to be the "source of truth" for Markdown, including lazy-loaded `SyntaxHighlighter` and `Playground`.
  - Replace the local `ReactMarkdown` implementation in `Blog.tsx` with `SharedMarkdown`.
  - Ensure `SharedMarkdown` handles `react-playground` language correctly for both Chat and Blog.

### 2.3 Unified Skeleton Loaders

- **Requirement:** Standardize the "Loading" experience across the application.
- **Action:**
  - Audit `Skeleton.tsx` and ensure it has necessary variants (text, rectangular, circular).
  - Create a `PageLoader` component that uses these skeletons for a consistent look.
  - Implement `Suspense` fallbacks in `App.tsx` for route transitions.
  - Standardize the `if (isLoading)` blocks in pages to use the new `PageLoader` or specific skeleton layouts.

## 3. Acceptance Criteria

- [ ] No direct `fetch("/api/v1/...")` calls remain in page components (excluding external APIs like Web3Forms if applicable).
- [ ] `Blog.tsx` uses `SharedMarkdown`.
- [ ] Navigation between pages shows consistent loading states.
- [ ] Build passes and no regressions in Chat or Blog rendering.
- [ ] All tests pass.

## 4. Constraints

- Do not introduce new heavy dependencies (e.g., React Query) unless specifically asked.
- Maintain existing i18n and theme support.
