# Technical Contract: v0.2.1 Post-Audit Refinements

- **Requester:** Orchestrator
- **Status:** Pending Implementation
- **Version:** 0.2.1

## 1. Overview

This contract addresses the remaining tasks for Version 0.2.1, focusing on performance optimization for the Blog Playground and UX enhancements for the Admin Logout action.

## 2. Blog Playground Optimization

The `@codesandbox/sandpack-react` library is heavy. While currently lazy-loaded, it can be further optimized to delay rendering and resource-intensive iframe loading until the user is actually viewing the playground.

### 2.1 Implementation Details

- **Component:** `frontend/src/components/blog/Playground.tsx`
- **Strategy:**
  - Implement `inView` logic using `IntersectionObserver`.
  - Only render the `Sandpack` component when the component is in or near the viewport.
  - Provide a high-fidelity placeholder (Skeleton with specific height) while the component is not in view to prevent layout shifts.
  - Ensure the `Sandpack` component is only imported when needed if possible, or leverage the existing lazy loading at the parent level.

## 3. Admin Logout UX

Improve the user experience of the logout action in the administrative dashboard.

### 3.1 Implementation Details

- **Component:** `frontend/src/components/admin/AdminHeader.tsx`
- **UI Changes:**
  - Add hover and active styles to the logout button.
  - Recommended Tailwind classes: `hover:bg-error/10 hover:text-error active:scale-95 transition-all`.
- **Component:** `frontend/src/pages/Admin.tsx`
- **Logic Changes:**
  - Add a toast notification `toast.success("Logged out successfully")` within the `handleLogout` function.
  - Ensure `hapticFeedback` is still triggered.

## 4. Verification Plan

- **Blog Playground:**
  - Verify that `Sandpack` components are not rendered in the DOM until the playground enters the viewport.
  - Verify that the layout remains stable with the placeholder.
- **Admin Logout:**
  - Verify hover and click visual feedback.
  - Verify that a success toast appears upon logout.
- **Regression:**
  - Ensure existing admin functionality and blog rendering remain intact.
