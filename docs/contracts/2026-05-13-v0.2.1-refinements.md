# Technical Contract: v0.2.1 Post-Audit Refinements

**Status:** Proposed
**Version:** 0.2.1
**Owner:** Gemini CLI (Orchestrator)

## 1. Overview

This contract outlines the final refinements for Version 0.2.1, focusing on backend security architecture, API redundancy reduction, and frontend loading state unification using React 19 primitives.

## 2. Technical Requirements

### 2.1 Global Admin Security

- **Backend**: Apply `verify_admin_password` as a global dependency in `backend/routers/admin/router.py`.
- **Cleanup**: Remove local `Depends(verify_admin_password)` from all sub-routers in `backend/routers/admin/*.py`.
- **Verification**: Ensure all admin endpoints still return 401/403 if the token is missing or invalid.

### 2.2 Admin API Consolidation

- **Redundancy Removal**: Delete `GET` endpoints in `admin/projects.py`, `admin/experience.py`, and `admin/about.py` that duplicate public functionality.
- **Frontend Sync**: Update `ProjectsManager.tsx`, `ExperienceManager.tsx`, and `AboutManager.tsx` to fetch initial data from public endpoints (`/api/v1/projects`, etc.).
- **Exception**: `admin/blog.py` `GET` remains, as it must return unpublished posts.

### 2.3 Unified Loading State (React 19)

- **Migration**: Refactor `Blog.tsx` and `Projects.tsx` to use the React 19 `use()` hook for data fetching.
- **Cleanup**: Remove `isLoading` states and manual `<PageLoader />` branches.
- **Global Suspense**: Ensure the `Suspense` fallback in `App.tsx` correctly handles the loading state during data fetch.

## 3. Success Criteria

- [ ] Admin section remains fully functional and secure.
- [ ] Redundant backend code is removed.
- [ ] `Blog` and `Projects` pages display the global `PageLoader` via `Suspense` during data fetch.
- [ ] No regressions in public data display.
