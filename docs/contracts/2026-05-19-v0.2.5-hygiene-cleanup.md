# Technical Contract: V0.2.5 Codebase Hygiene & Audit Resolution

- **Requester:** Orchestrator
- **Status:** Completed (2026-05-19)
- **Version:** 0.2.5

## Scope

### 1. Frontend Dead Code Cleanup

- **Target**: `frontend/lib/api.ts`
- **Action**: Remove `fetchCMS` and `updateCMS` functions.
- **Reason**: These were intended for unified Admin API access but were bypassed by individual admin components using direct `fetch()` calls.

### 2. Backend Middleware Optimization

- **Target**: `backend/middleware.py`
- **Action**: Remove the redundant `if hasattr(response, "background")` block.
- **Reason**: The block contains only comments and a `pass` statement, adding unnecessary visual noise.

### 3. Dependency Hardening

- **Target**: `backend/pyproject.toml`
- **Action**:
  - Remove `google-auth-oauthlib`.
  - Remove `requests` (implicit dependency).
- **Reason**: `google-auth-oauthlib` is unused (Drive uses refresh tokens), and `requests` is pulled in by other libraries.

### 4. Test Cleanup

- **Target**: `frontend/src/App.test.ts`
- **Action**: Remove the boilerplate math test.
- **Reason**: Preparing for meaningful UI component testing.

### 5. Data Decoupling Strategy

- **Target**: `backend/responses/*.py`
- **Action**: Refactor hardcoded response dictionaries into JSON or YAML fixtures.
- **Reason**: Improves maintainability by separating static persona data from application logic.

## Acceptance Criteria

- [ ] `frontend/lib/api.ts` no longer contains `fetchCMS` or `updateCMS`.
- [ ] `backend/middleware.py` is cleaner without the redundant `hasattr` check.
- [ ] `backend/pyproject.toml` reflects the pruned dependencies and `uv sync` has been run.
- [ ] `frontend/src/App.test.ts` contains only relevant tests (or is empty if no other tests exist).
- [ ] Response data is successfully loaded from external fixtures in `backend/responses/`.
- [ ] All tests pass.
