# Technical Contract: V0.2.5 Backend Hardening

- **Requester:** Orchestrator
- **Status:** Completed (2026-05-19)
- **Version:** 0.2.5

## Scope

### 1. In-Memory Task Queue for Middleware

- **Target**: `backend/middleware.py`
- **Action**:
  - Implement a singleton `TaskQueue` using `asyncio.Queue`.
  - Create a background worker (started via FastAPI lifespan) that consumes from the queue.
  - Update `VisitorTrackingMiddleware` to push logs to the queue instead of creating unmanaged `asyncio.tasks`.
  - **Reason**: Improves event loop predictability and prevents potential resource exhaustion from unmanaged tasks during traffic spikes.

### 2. Sanitized & Environment-Driven CORS

- **Target**: `backend/config.py` and `backend/main.py`
- **Action**:
  - Add `CORS_ORIGINS: List[str]` to `Settings`.
  - In `main.py`, parse `settings.CORS_ORIGINS`.
  - Ensure `http://localhost:5173` is only included in `development` mode or if explicitly provided.
  - Clean up `allow_origins` to strictly use the configured list.
  - **Reason**: Enhances security by strictly controlling which domains can interact with the API.

## Acceptance Criteria

- [ ] `VisitorTrackingMiddleware` no longer uses `asyncio.create_task` directly for logging.
- [ ] A background worker process logs visitor data from the queue.
- [ ] CORS `allow_origins` is dynamically populated from environment variables.
- [ ] Backend starts without errors and continues to log visitors correctly.
- [ ] `uv run pytest` passes.
