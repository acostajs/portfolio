# Technical Contract: Feedback & Analytics Expansion

- **Requester:** Orchestrator
- **Status:** Pending Implementation
- **Version:** 0.2.0

## 1. Overview

This contract defines the enhancements for the chatbot feedback system and the administrative analytics dashboard. The goal is to identify specific areas of the chatbot's knowledge (modules/categories) that are underperforming based on user feedback.

## 2. Shared Schema Updates

### 2.1 ChatFeedback Model (Backend & Frontend Types)

The `ChatFeedback` schema will be expanded to include context about the matched response.

```json
{
  "user_message": "string",
  "assistant_reply": "string",
  "is_helpful": "boolean",
  "module": "string (optional)",
  "category": "string (optional)"
}
```

## 3. Backend Enhancements

### 3.1 Data Model

- Update `ChatFeedback` in `backend/models.py` to include `module` and `category` fields.

### 3.2 API Endpoints

- **POST `/api/v1/chat/feedback`**: Update to accept `module` and `category`.
- **GET `/api/v1/admin/analytics/feedback`**: Ensure it returns the new fields.

### 3.3 Logic

- Refactor `save_chat_feedback` background task in `backend/routers/chat.py` to handle the new fields.

## 4. Frontend Enhancements

### 4.1 Chat Feedback Loop

- Update the feedback submission logic in `BotMessage.tsx` (or wherever the thumbs are) to include the `module` and `category` associated with the response.
- _Note:_ The `chat` API response should probably include the `module` and `category` of the matched trigger to make this possible.

### 4.2 Admin Analytics Dashboard

- **Negative Feedback Trends**: In `AnalyticsManager.tsx`, implement a breakdown section showing which modules/categories have the most "Not Helpful" ratings.
- **Visuals**: Add a list or small chart highlighting "Weakest Triggers" (high negative ratio).

## 5. Verification Plan

- **Backend**: Run `pytest` to verify the updated feedback endpoint.
- **Frontend**: Manual test of submitting feedback and checking the Analytics dashboard.
- **Data Integrity**: Verify that `module` and `category` are correctly saved in the database after a feedback event.
